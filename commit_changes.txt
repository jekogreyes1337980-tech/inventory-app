======================================================
COMMIT 2: "Migrate backend to use SQLite"
======================================================

NEW FILES CREATED:
------------------

1. server.js (Node.js + Express backend)
   - GET  /api/data/:key  → reads a key from SQLite database
   - POST /api/data/:key  → writes/updates a key in SQLite database
   - POST /api/reset      → wipes all data from SQLite database
   - Serves all static files (HTML, CSS, JS) from the project folder
   - Runs on http://localhost:3000

2. package.json
   - Lists dependencies: express, sqlite3, sqlite, cors
   - Run "npm install" to install them

3. data.db (auto-created on first run)
   - SQLite database file
   - Stores all inventory data in a single table: store(key, value)

======================================================
WHAT THIS MEANS FOR THE PROJECT
======================================================

BEFORE these commits:
- All data was saved in browser localStorage (single device only)
- Everything was in one giant app.js file
- No server, just open index.html in a browser

AFTER these commits:
- Data is saved in a real SQLite database file (data.db)
- Multiple users/devices can share the same data
- Code is split into clean modules (easier to work on)
- Must run "node server.js" and open http://localhost:3000
